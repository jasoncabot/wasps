# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## About

Wasps is a browser-based card game — a custom variant of Crazy Eights/Mau Mau. Single-repo: React+Phaser 3 frontend in `src/`, Cloudflare Worker backend in `worker/`, built and served together via Vite + `@cloudflare/vite-plugin`.

## Commands

```bash
npm run dev          # Vite dev server + local Worker (miniflare)
npm run build        # Production build → dist/client/ (SPA) + worker bundle
npm run deploy       # wrangler deploy to Cloudflare
npm run cf-typegen   # Regenerate worker-configuration.d.ts from wrangler.jsonc
npm test             # Run worker tests via vitest (cloudflare pool)
npm run test:watch   # Watch mode
```

## Architecture

### Frontend (`src/`)

The frontend is a **React + Phaser 3** hybrid. React renders the shell and notifications; Phaser owns the game canvas.

Entry: `src/index.tsx` → `src/components/App.tsx` mounts `<GameContainer />` which boots the Phaser game. `PreloadScene` loads assets, then hands off to `MainScene`.

**Core game layer** (`src/scenes/objects/`):

- **`Card.ts`** — All card types (`CardSuit`, `CardRank`, `Card`), plus pure functions for card logic (`isSpecial`, `forcesPickup`, `changesSuit`, `validatePlay`, `pack`). Card image assets are numbered 1–52 via `card_N = (suit-1)*13 + rank`; joker is `card_joker`, face-down is `card_back`.
- **`Game.ts`** — Pure game state machine. Holds all hands, draw pile, play history, forced pickup count, and direction. Exposes `deal()`, `applyTurn()`, `processTurn()`, `changeTurns()`. No knowledge of rendering or which player is "me."
- **`TurnController.ts`** — Bridges `Game` and the view. Knows which player is "me" and translates absolute player indices into relative indices (you are always index 0). Implements both `GameEventHandler` (called by `Game`) and `TurnController` (called by `Game.processTurn`). Pre-computes the AI move at turn start so it's available immediately.
- **`TurnBuilder.ts`** — Stateless AI logic. `calculateTurn(context)` does a depth-first search over all valid card combinations and picks the longest play. `validatePlay` / `isValidPlay` are also used by the UI for move validation.
- **`Player.ts`** — Just `{ name: string }`.
- **`MainScene.ts`** — Phaser `Scene` implementing `ViewEventHandler`. Handles all rendering (cards, names, suit chooser, pickup pile) and user input. Stores a `storedTurnCallback` promise resolver fulfilled when the human player acts. Dispatches `game-notification` CustomEvents for errors, which `App.tsx` displays as toasts.

**Turn flow:**

1. `TurnController.startGame()` calls `game.deal()` then loops `game.processTurn()`
2. `game.processTurn()` calls `turnController.onTurnStarted()` → `viewEventHandler.onTurnStarted()` → returns a `Promise<TurnCommand>`
3. For human turns, `MainScene` resolves the promise via `storedTurnCallback` when the player taps a card or the pickup pile
4. For AI turns, `MainScene` resolves immediately with the pre-computed AI promise
5. `game.applyTurn()` mutates game state; `game.changeTurns()` advances the turn order

### Worker (`worker/`)

`worker/index.ts` is a Cloudflare Worker that serves the built SPA via the `ASSETS` binding. The `Env` interface is generated by `npm run cf-typegen` into `worker-configuration.d.ts`.

### Build

Vite bundles `src/` into `dist/client/` and the worker separately. The `cloudflare()` plugin reads `wrangler.jsonc` to locate `worker/index.ts`. Phaser is split into its own chunk via `manualChunks`. `assetsInlineLimit: 0` prevents card PNGs from being base64-inlined (which would break Phaser's XHR-based image loader).

### Special card rules (encoded in `Card.ts`)

| Card             | Effect                                                                     |
| ---------------- | -------------------------------------------------------------------------- |
| Ace              | Play on anything; chooser changes suit                                     |
| 2                | Next player picks up 2 (stackable)                                         |
| Black Jack (♣/♠) | Next player picks up 7 (stackable)                                         |
| Red Jack (♥/♦)   | Cancels a pending Black Jack pickup                                        |
| Queen            | Reverses direction (odd count reverses, even cancels)                      |
| Joker            | Play on anything; next player picks up 4; chooser changes suit (stackable) |
